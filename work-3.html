Overview

This project was inspired by classic 2D arcade airplane shooters, but my main focus was not just visual style. I used the game format as a way to explore structured interaction logic, changing conditions, and system behavior over time.

The player fights waves of enemy planes approaching from multiple directions. Once a score threshold is reached, regular enemies stop spawning and the experience shifts into a boss fight. The boss then changes speed and attack patterns based on health, creating a second gameplay phase and making the logic of the system visible through play.

What I Designed

State-Based Progression

Structured the gameplay around changing states, from regular enemy waves to boss encounter, so the whole game evolves through clear conditional logic.

Boss Phase Logic

Designed a two-phase boss system where attack behavior and movement shift with health, making progression feel dynamic instead of repetitive.

Interactive System Design

Used score, spawning rules, enemy types, and boss triggers together to build a gameplay loop that turns programming logic into a playable experience.

Process

The project began with the familiar structure of a simple shooter game, but quickly became an exploration of how game states can create pacing and variation. I focused on building clear conditions: when enemies appear, when they stop, when the boss is triggered, and how the fight changes after that.

Rather than treating the project as only a visual remake of a pixel shooter, I used it to study how logic, rules, and timing can shape the player's experience. The final result is a small arcade game that highlights both interactive play and the design of system behavior.
