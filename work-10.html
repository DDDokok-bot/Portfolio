Overview

This project is a puzzle-based Unity game that more deeply explores condition design in gameplay. The core idea is that different conditions trigger different behaviors and results, so progression depends on understanding how actions, objects, and rules connect across each level.

The game moves through three stages. Players begin as a sheep, solve an environmental activation puzzle, make a correct transformation choice in the second level, and finally complete a multi-step escape puzzle in a dark room. The project focuses less on action and more on how layered logic can shape progression.

What I Designed

Condition-Based Progression

Built each stage around specific trigger conditions so players must satisfy the right sequence of actions before the next event or pathway becomes available.

Multi-Stage Puzzle Flow

Structured the game across three distinct puzzle phases, each with a different type of challenge: environmental activation, rule-based choice, and spatial object interaction.

Level Logic Design

Focused on how level layout, object placement, and hidden requirements could guide discovery and make the puzzle logic readable through play.

Level 1

The game begins with the player controlling a sheep character in the first level. To progress, the player must eat all of the white flowers in the grass. Completing that condition wakes the giant duck in the pond and triggers the appearance of stairs leading to the next stage.

This opening level introduces the project’s logic clearly: the player is not simply moving forward, but learning that the environment changes only when the correct condition is fulfilled.

Level 2

In the second level, the player encounters three doors, each allowing transformation into a different animal. The environment indicates that only one of those animal forms can continue to the next level, so the player must interpret the condition correctly rather than choosing at random.

After making the correct choice, the player is allowed to enter the next area through the appropriate door. This stage focuses on conditional selection and using environmental cues to support puzzle reasoning.

Final Level

The final level takes place in a dark room with a locked door. The player must first find the light switch, then locate the key hidden inside a glass of water. After breaking the glass and retrieving the key, the player still cannot directly reach the lock because the character model is too small.

To complete the puzzle, the player must also move a nearby box into the correct position, climb onto it, and jump in order to reach the lock and open the door. This final stage combines object interaction, spatial reasoning, and multiple dependent conditions into one sequence.

Process

This project was developed in Unity as a deeper study of condition design in games. Instead of focusing only on atmosphere or character movement, I used the game format to test how different rules, triggers, and dependencies could create varied puzzle experiences from one level to the next.

The result is a puzzle game prototype where progression depends on understanding how conditions unlock new behaviors, objects, and routes. Each level explores a different kind of game logic while still contributing to one continuous progression structure.
